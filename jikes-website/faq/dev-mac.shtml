Jikes on Mac

Jikes has been successfully compiled and used on MacOS X Public Beta. We don't know about how Jikes is built on other versions of this OS, but if you do have additional info, please drop a post in the Jikes or Jikes-Dev mailing list so we can update this topic.

Development tools

You'll need at list a subset of the Developper Tools offered by Apple to be able to build your own Jikes. You won't probably need the entire suite (especially the GUI part), but you'll definitely need the GNU tools that are part of it.

Download the Dev Tools from Apple:
http://developer.apple.com/tools/projectbuilder/#Download

Source control

You can either decide to build Jikes based on the release source tree, or decide that you want to get more serious about Jikes and use CVS. The Project Builder comes with a bundled CVS, so you don't need an additional tool.

NEEDED: Additional info on how to use the CVS from Project Builder is extremely important but missing here. Please contact us if you think you can help us improve these instructions.

Here is what you need to do if you choose to use the release source tree:

Download the Jikes source distribution:
http://oss.software.ibm.com/developerworks/project/showfiles.php?group_id=10
Then, in a command window, uncompress the archive:
gnutar -xzf [jikes-archive.tar.gz)]

After these steps, a jikes directory must be available, containing the Jikes source files.

Build environment configuration

Now that you have a Jikes source tree, just have a look into it. Right in the jikes directory, you'll find a script named configure, which role is to automatically detect the features of your build environment and produce a Makefile and config.h files which contain the appropriate definitions and instructions for your platform (see the general Compiling section for additional info about the configure script).

Just run it from a command windows:

In a command window:
cd [jikes directory]
./configure

Compiling

Your build environment is now setup and ready. In the same command window, run the command below:

make

When the compilation is finished, your Jikes executable must be found in the src directory. Congratulations!

Building Jikes distributables

You may want to generate a distributable package for your newly compiled Jikes. The generated Makefile provides the adequate targets: install and install-strip. The install-strip target will remove any additional symbols included by the compiler for debugging, making the executable more compact. If you intend to develop on Jikes (thus probably debug it), use the install target.

su - root

For development:
make install

Or for day-to-day use:
make install-strip
